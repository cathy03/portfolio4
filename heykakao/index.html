헤이카카오 개요
헤이카카오 앱은 카카오미니 기기 설정 서비스에서 음성, 시각 인식 등 다양한 AI 기술을 활용한 라이프 어시스턴스 서비스로의 전환을 목표로 했습니다. 이에 따라, 각 기능이 명확하게 인지되고 쉬운 사용성을 갖춘 프로덕트를 만들기 위해 프로토타이핑을 통한 다양한 테스트를 진행했습니다.

1. Main Interaction Design
구글이 메인 화면에 검색 필드만 노출하여 핵심 기능에 집중하도록 한 것처럼, 헤이카카오도 음성, 음악, 카메라 검색 등 각 기능에 집중할 수 있도록 구성했습니다. 또한, 헤이카카오의 브랜드 아이덴티티인 노란 원을 활용해 디자인을 완성했습니다.
Type A

카카오미니 기기와 같이 어두운 톤의 디자인으로, 원이 퍼져나가는 애니메이션으로 각 기능을 인지할 수 있도록 하였습니다.

Type B

기존 카카오미니 사용자가 앱에서 기기 관리에 익숙함을 유지하면서, 새로운 기능도 쉽게 파악할 수 있도록 대시보드 페이지를 구성했습니다.

2. Voice Interaction Design
음성 인식, 답변을 담당하는 보이스 에이전트 디자인 및 인터랙션 디자인을 하였습니다. 에이전트 외에도 화면 전환 효과와 색상(음성인식 시 어두운 화면, 대기 및 답변시 밝은 화면)을 통해 각 단계를 시각적으로 명확하게 구분할 수 있도록 디자인했습니다.
단답형 시나리오 인터랙션

손을 사용하기 어려운 상황에서 음성 입력 방식의 사용 빈도가 높아지는 점을 고려하여,
사용자가 잠깐 흘겨보더라도 상태를 명확하게 인지할 수 있도록 트랜지션 효과와 디자인을 적용했습니다.

슬롯필링 시나리오 인터랙션

에이전트가 여러 번의 질문 단계를 통해 답변을 도출하는 슬롯필링 시나리오를 위한 인터랙션 설계 및 디자인을 진행했습니다.

3. Music Recognition Interaction Design
음악 인식 기능의 화면 전환 효과는 음성 인식과 동일하게 유지하면서, 색상과 에이전트 인터랙션에 차이를 두어 음성 인식과 구분되도록 디자인했습니다. 또한, 음성 입력을 통해 음악 인식 기능으로 바로 전환되는 시나리오까지 고려하여 작업했습니다.
4. Translation Interaction Design
음성 입력과 타이핑 입력이 가능한 번역 기능이 기존 사용성과 자연스럽게 어우러질 수 있도록, 디자인 및 인터랙션을 프로토타입을 통해 검증했습니다.
텍스트 입력이 강조된 방향

다른 번역 서비스와 유사한 텍스트 입력 방식을 통해 익숙한 사용자 경험을 제공하는 시안입니다.

음성 입력이 강조된 방향

음성 입력이 메인인 헤이카카오 앱의 사용자 경험을 번역에도 일관되게 반영한 시안입니다.

5. Dictation Interaction Design
녹음한 내용을 텍스트로 변환하고 화자를 분석할 수 있는 받아쓰기 기능의 디자인 및 인터랙션 작업을 진행했습니다.
Type A

받아쓰기 기능 활성화 시, 화면 전환 효과를 음성 및 음악 인식과 동일하게 구성하여 서비스의 일관성을 유지했습니다.

Type B

음성 입력 방식이 주된 헤이카카오 서비스에 맞춰, 녹음과 목록을 분리하여 구성함으로써 서비스 일관성을 유지하는 방향으로 작업했습니다.
